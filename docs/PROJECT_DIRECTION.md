# 설탕과소금 프로젝트 방향

## 핵심 철학

설탕과소금은 이 세상에 꼭 필요한 설탕과 소금 같은 인재를 기르고 돕겠다는 개인적 인생관이자 경북직업교육연구회의 중요한 모토이다.

개인 홈페이지는 이 철학을 설명하는 장소이면서, 여러 독립 시스템으로 들어가는 포털 역할을 맡는다. 실제 기능은 한 홈페이지 안에 모두 섞지 않고 별도 프로젝트로 분리해 연결한다.

## 포털의 역할

- 홈 화면은 개인 이력보다 콘텐츠와 시스템 입구를 우선한다.
- 개인 소개, 활동 기록, 교직 이야기는 `주인장` 영역에 모은다.
- 직업계고 공채 정보, 법률정보, 전자책 서재, AI 도구, 기록 보관소는 독립 프로젝트로 확장한다.
- 각 시스템은 필요할 때 홈페이지 카드나 메뉴에서 연결한다.

## 직업계고 취업지도 허브의 방향

특성화고와 직업계고 취업 담당 선생님에게 가장 시급한 콘텐츠 중 하나는 흘러가는 채용정보를 놓치지 않도록 정리하고, 학생 지도를 위한 기업자료와 관련자료까지 함께 제공하는 취업지도 허브이다.

단톡방, 공식 채용공고, 공공기관 채용 사이트, 첨부 공고문에서 확인한 공개 가능한 정보만 공식 원문 링크 기준으로 구조화한다.

공채 정보는 과거 공고나 마감된 정보를 현재 유효한 정보처럼 보이게 해서는 안 된다. 모든 공고에는 확인일, 접수기간, 원문 링크, 마감 여부를 함께 표시하고, 원문 링크가 사라졌거나 내용이 바뀐 경우에는 `확인 필요` 상태로 낮춘다.

초기 핵심 항목은 다음과 같다.

- 기관명과 공고명
- 고졸채용, 고졸예정, 학교장 추천 등 지원 구분
- 채용인원, 직무, 근무지역
- 지원자격과 제한사항
- 접수기간과 마감일
- 필기, 면접, 합격자 발표, 임용예정일
- 접수 링크와 첨부자료
- 담당 교사용 확인 체크리스트
- 근로계약, 현장실습, 산업안전 등 법률정보 연결 필요 여부

공채 하나마다 함께 제공할 수 있는 확장 정보는 다음과 같다.

- 기업 또는 기관 개요
- 주요 사업, 직무, 근무지, 조직문화 등 학생 지도에 필요한 공개자료
- 직무기술서, 자기소개서 문항, 면접 대비 자료, NCS 자료
- 학교장 추천, 현장실습, 채용형 인턴 등 유형별 유의사항
- 담당 교사용 일정표와 학생 안내문
- 전자책 서재의 관련 강의자료, 취업지도 자료, 면접자료 연결
- AI 법률정보 도우미의 근로계약, 현장실습, 산업안전, 개인정보, 차별금지 관련 원문 연결

이 허브는 특정 단톡방을 복제하는 서비스가 아니라, 학교 현장에서 공개 채용정보와 지도자료를 더 빨리 찾고 비교할 수 있게 정리하는 게시판 또는 앱으로 설계한다.

## AI 법률정보 도우미의 방향

첫 시스템은 특성화고등학교 학생, 선생님, 학부모, 학교 구성원의 취업과 학교 현장 문제를 돕는 법률정보 시스템으로 준비한다.

초기 핵심 주제는 다음과 같다.

- 특성화고 취업 지원
- 도제학교
- 현장실습
- 해외 현장실습
- 중대재해처벌법
- 학교폭력
- 정규 교사와 기간제 교사
- 행정실 정직원과 상근 기간제 행정직원
- 학생관리
- 민원 대응
- 선생님들의 개인적 법률정보 지원

이 시스템은 법률 자문이나 사건 판단이 아니라 공개 법령, 판례, 행정자료, 원문 근거를 찾기 쉽게 정리하는 정보 제공 도구로 표현한다.

AI는 기억이나 추측으로 답하지 않고, 검색된 공식 원문과 승인된 API 결과만 요약한다. 원문 근거가 없거나 서로 충돌하는 경우에는 답을 단정하지 않고 `확인 필요`로 안내한다.

법률정보는 법령 제정, 개정, 시행일, 부칙, 위임 행정규칙 등 변동 정보를 계속 확인한다. 사용자가 보는 법률정보에는 현행 여부, 시행일, 개정일, 확인일, 원문 링크를 함께 표시한다.

## 정보 신뢰성 원칙

설탕과소금의 공채 정보와 법률정보 시스템은 빠른 답변보다 현재성, 원문성, 검증 가능성을 우선한다.

- 공식 원문, 공공 API, 기관 채용 사이트를 1차 출처로 사용한다.
- 사용자에게 보이는 모든 핵심 정보에는 출처명, 원문 링크, 확인일을 함께 표시한다.
- 공채 정보는 접수기간과 마감 여부를 자동 계산하고, 마감된 공고는 현재 모집처럼 노출하지 않는다.
- 법령, 판례, 행정자료는 검색 시점과 원문 기준을 함께 저장한다.
- 법령 정보는 현행 여부, 제정·개정일, 시행일, 확인일을 함께 표시한다.
- 기업자료는 공식 홈페이지, 공공기관 경영공시, 채용공고 첨부자료 등 공개 출처를 우선 사용한다.
- 전자책 서재와 연결되는 자료도 출처, 작성일, 최신 확인일을 관리한다.
- API 응답, 원문 링크, 첨부파일 정보가 서로 다르면 `확인 필요` 상태로 표시한다.
- AI 요약은 원문 뒤에 붙는 보조 설명으로만 사용하고, 원문보다 앞서 단정하지 않는다.
- 출처가 없는 답변, 날짜가 불명확한 답변, 현재 유효성을 확인하지 못한 답변은 제공하지 않는다.
- 캐시를 사용하더라도 최신 확인 시각과 만료 기준을 둔다.
- 중요한 사안은 전문가, 관계 기관, 공식 담당자 확인을 권장한다.

시스템 구현 시에는 `최신 원문 확인 -> 구조화 -> AI 요약 -> 출처 표시 -> 마감/변경 경고` 순서가 무너지지 않도록 한다.

## 교육 전자책 서재의 방향

교육강의자료실은 단순 자료 목록이 아니라 전자책 서재 시스템으로 발전시킨다.

- 강의 자료
- 강연 자료
- AI 활용 수업 자료
- 교육자료실
- 교육정보 나눔
- 경북직업교육연구회 활동 자료

사용자가 자료를 모아 읽고, 주제별로 찾고, 나중에는 전자책처럼 열람할 수 있는 독립 시스템으로 설계한다.

## 확장 기준

- 새 기능은 설탕과소금의 철학과 교육 현장 지원 목적에 맞는지 먼저 확인한다.
- 포털은 입구, 독립 프로젝트는 실제 기능이라는 구조를 유지한다.
- 법률정보 시스템에서는 항상 원문 근거와 정보 제공 고지를 우선한다.
- 전자책 시스템은 기존 보안 전자책 프로젝트와 섞지 않고 새 프로젝트로 검토한다.
- Firebase, API 키, 배포, 새 저장소 생성은 실행 전 사용자 확인을 받는다.
