# 특성화고 채용정보 자동 수집 시스템 계획

## 원칙

- 수동 등록은 기본 운영에서 제외한다.
- 공식 API, 공공데이터, 공식 RSS, 공식 제휴 경로를 우선 사용한다.
- 무단 크롤링은 약관, robots, 저작권, 개인정보 위험을 확인하기 전까지 기본 수집 경로로 사용하지 않는다.
- 모든 공고는 원문 링크, 마감일, 검증 시각, 출처 상태를 함께 저장한다.
- 서비스의 중심은 대기업, 공공기관, 군, 정부·비영리기관 등 공식 공채다.
- 공채는 공고 게시 첫날 수집을 최대 과제로 삼고, 늦게 발견된 공고는 누락 점검 대상으로 표시한다.
- 공채는 회사·기관 자체 홈페이지 또는 채용대행 공식 공지사항을 반드시 2중 확인한 뒤 상세 정보로 제공한다.
- 필기시험 없는 채용은 중견기업 중심으로 핵심 정보만 간단히 제공하고, 세부 조건은 회사 공식 공고 또는 인사담당자 확인을 안내한다.
- 공채 원서 접수가 끝난 채용건은 제목 끝에 `(원서 마감)`을 붙여 지원 가능 상태를 혼동하지 않게 한다.
- API 키, 토큰, 접근키는 GitHub Actions Secrets에만 저장하고 저장소에는 기록하지 않는다.
- 필기시험 여부는 단정하지 않고 `confirmed`, `likely`, `not_found`, `unknown`으로 분리한다.

## 자동화 구조

`tools/fetch_vocational_jobs.mjs`가 하루 3회 실행되어 `assets/job-feed.json`을 생성한다.

- 실행 시간: 09:10, 14:10, 19:10 KST
- 실행 위치: GitHub Actions
- 저장 방식: 정적 JSON 자동 갱신 후 커밋
- 표시 방식: `index.html`이 `assets/job-feed.json`을 읽어 홈페이지 공채 허브에 표시
- 실행 전 `node --check tools/fetch_vocational_jobs.mjs`로 문법 오류를 먼저 차단한다.

## 제공 화면 구분

1. 필기·공식전형 공채
   - 대기업 공채, 공공기관, 잡알리오, 나라일터, 군 부사관, 군무원, 정부·비영리기관 채용
   - 필기, NCS, 직업기초능력, 인적성, 체력검정, 공개경쟁 등 키워드를 우선 감지한다.
   - 회사·기관 공식 채용 공지사항 또는 채용대행 공식 공고 URL을 확보하고, 가능하면 해당 페이지 본문에서 회사명·공고명·전형 키워드를 대조한다.
   - 상세 카드에는 전형절차, 응시자격, 마감일, 공식 공고 확인 상태, 학교 지도 체크 포인트를 함께 표시한다.
   - 원서 접수 마감 이후에도 필기·면접 일정 확인이 필요한 공채는 일정 기간 `원서 마감` 상태로 유지한다.

2. 면접중심·현장형 채용
   - 중견기업, 채용연계, 현장실습형, 수시·상시 채용
   - 필기시험이 확인되지 않은 공고는 상세 해설 대상이 아니라 간단 안내 대상으로 분류한다.
   - 근무조건, 급여, 제출서류, 청소년·미성년자 근로 가능 여부는 회사 공식 공고 또는 인사담당자에게 확인하도록 안내한다.

## API 키와 공식 피드 연결 준비

API 키나 공식 피드 URL이 확보되면 저장소 파일에 값을 쓰지 않고 GitHub Actions Secret에만 저장한다. 수집기는 Secret 값은 출력하지 않고, `secretReadiness`에 Secret 이름과 준비 상태만 기록한다.

선생님이 직접 해야 할 일은 [채용정보 API 키 연결 쉬운 순서](JOB_FEED_SECRET_SETUP_SIMPLE.md)에 따로 정리한다.

- `ready`: 즉시 수집 시도 가능
- `partial`: 키 또는 URL 중 일부만 들어와 추가 설정 필요
- `missing`: 아직 Secret 또는 공식 피드 경로 없음

인사혁신처 공공취업정보는 공공데이터포털 키로 국가기관, 지방자치단체, 시도교육청, 공공기관 채용 후보를 확인한다. 기본 Secret은 `MPM_PUBLIC_JOB_SERVICE_KEY`이고, 공통 공공데이터포털 키를 쓰는 경우 `DATA_GO_KR_SERVICE_KEY`도 읽는다.

기재부 공공기관 채용정보는 공공기관 채용공시 목록을 구조화해 잡알리오 축을 보강한다. 기본 Secret은 `MOEF_PUBLIC_RECRUIT_SERVICE_KEY`이고, 공통 공공데이터포털 키를 쓰는 경우 `DATA_GO_KR_SERVICE_KEY`도 읽는다.

서울특별시교육청 하이잡 채용정보는 별도 키 없이 공식 공개 목록과 상세 원문을 전용 어댑터로 확인한다. 매칭데이·주요행사 게시판은 공식 구조를 추가 확인한 뒤 같은 방식으로 확장한다.

나라일터, 군 채용, 잡코리아, 인크루트, 알바몬, 알바천국, 기타 지역 교육청, 비영리기관은 `*_API_URL` 또는 `EDU_JOB_CENTER_FEEDS`가 들어오면 범용 공식 피드 어댑터가 먼저 JSON/XML/RSS 구조를 읽어 정규화를 시도한다. API 문서가 확보되면 이후 전용 어댑터로 바꿔 정확도를 높인다.

공식 URL Secret 안에 키를 넣어야 하는 서비스는 URL에 `{API_KEY}`, `{KEY}`, `{SERVICE_KEY}` placeholder를 둘 수 있다. 수집기는 실행 중에만 placeholder를 실제 Secret으로 치환하고, 생성 JSON에는 Secret 값이 남지 않게 한다.

## 정규화 필드

각 공고는 다음 필드로 정규화한다.

- 원문 출처와 원문 링크
- 기업명, 공고명, 지역, 고용형태
- 학력조건, 경력조건
- 마감일과 마감 상태
- 공식 채용 소스 원문 `sourceOfficialUrl`
- 회사·기관 공식 채용 공지사항 `companyNoticeUrl`
- 공식 원문 우선순위 `primaryOfficialUrl`, `officialNoticePriority`
- 공식 공고 2중 확인 상태 `sourceVerification`
- 표시 정책 `servicePriority`, `detailLevel`
- 첫날 수집과 누락 점검 상태 `collectionAudit`
- 고졸·특성화고 적합도 점수
- 학교장 추천 여부
- 전형 트랙: `exam-formal` 또는 `direct-interview`
- 필기시험 판단: `confirmed`, `likely`, `not_found`, `unknown`
- 분야: 공공기관, 정부·공무직, 군, 비영리, 대기업, 중견·중소, 시간제 등
- 법률·지도자료 체크 태그
- 검증 시각

## 회사 공고 2중 확인 규칙

공채 상세 서비스는 아래 순서로 확인한다.

1. 잡알리오, 고용24 공채속보, 나라일터, 군 채용 등 공식 채용 소스에서 공고를 발견한다.
2. 해당 공고 안의 회사·기관 공식 채용 공지사항 또는 채용대행 공식 공고 URL을 추출한다.
3. 가능하면 공식 공지 페이지에 접속해 회사명, 공고명 주요 단어, 채용·모집·전형 키워드를 대조한다.
4. 확인 결과를 `sourceVerification.doubleCheckStatus`에 저장한다.

상태값은 다음과 같다.

- `company_notice_confirmed`: 회사·기관 공지 본문에서 공고 맥락이 확인됨
- `company_notice_reachable`: 회사·기관 공지 페이지 접속은 확인됨
- `company_notice_linked`: 공식 소스에서 회사·기관 공지 링크를 확보했지만 본문 대조는 추가 필요
- `company_notice_required`: 공채 후보지만 회사·기관 공식 공지 확인이 필요함
- `company_contact_recommended`: 필기 없는 채용으로 회사 확인 안내 대상

`detailed-public-recruit` 상태가 아닌 공채는 화면에서 “공채 상세 확인중”으로 표시하고, 상세 확정 정보처럼 과장하지 않는다.

## 첫날 수집과 누락 점검

공채 수집기는 각 공고의 게시일과 최초 발견일을 비교해 `collectionAudit`을 기록한다.

- `firstDayCollected: true`: 공고 게시 첫날 자동 수집
- `firstDayStatus: next_day`: 다음 날 발견되어 점검 필요
- `firstDayStatus: late_detected`: 2일 이상 늦게 발견되어 누락 점검 필요
- `missedReviewNeeded: true`: 게시일 미확인 또는 첫날 수집 실패로 운영자가 소스 경로를 점검해야 함

현재 잡알리오 공식 공개 목록은 최대 40건까지 살펴보며, 향후 나라일터·군 채용·대기업 채용대행 공고도 같은 방식으로 첫날 수집 여부를 기록한다.

자동 수집 결과에는 `collectionReview` 운영 리포트를 함께 저장한다.

- `missedReviewItems`: 게시 첫날 수집하지 못했거나 게시일 확인이 필요한 공채
- `officialNoticePendingItems`: 공채 상세 대상이지만 회사·기관 또는 채용대행 공식 공고 확인이 부족한 공채
- `sourceGaps`: Secret, API 승인, 어댑터 구현, 공식 피드 확인이 필요한 미연결 소스
- `nextActions`: 누락 점검, 공식 공고 보강, 미연결 소스 확인 순서

## 필터링 기준

유지한다.

- 고졸, 고등학교, 특성화고, 직업계고, 마이스터고, 졸업예정, 학교장 추천이 명확한 공고
- 학력무관 공고는 고졸·졸업예정자가 응시할 수 있는 것으로 우선 보되, 아래 제외 조건에 해당하는지 함께 검증한다.
- 신입·경력무관·청년인턴·채용형 인턴·공무직·업무지원직·기간제·9급·지역인재·기능인재처럼 고졸 지원 가능성이 높은 공고
- 원문 링크와 마감일 확인이 가능한 공고
- 군 부사관, 군무원, 공공기관, 비영리기관 중 고졸 지원 가능성이 있는 공고

제외한다.

- 필기·공식전형 공채가 아닌 이미 마감된 공고
- 원서 마감 후 21일이 지난 공채
- 대졸 이상, 대졸(4년), 전문대졸, 4년제 이상, 석사·박사 등 고졸 예정자와 맞지 않는 조건만 있는 공고
- 원장·센터장·기관장·본부장·개방형 직위·전문계약직·연봉계약직·임원성 직위처럼 고등학교 졸업생 채용 코너와 맞지 않는 공고
- 학력무관이더라도 전문면허·전문자격 전용 또는 경력 전용으로 보이고 고졸·신입 가능 신호가 없는 공고
- 원문 링크가 없거나 채용 여부 확인이 어려운 공고

## 공식 자료 확인

- 인사혁신처 공공취업정보 조회 서비스는 나라일터와 연계된 공공취업정보를 실시간 XML로 제공하며, 국가기관·지방자치단체·시도교육청·공공기관 채용 후보를 확인하는 축이다.
- 기재부 공공기관 채용정보 조회서비스는 공공기관 채용공시 목록·상세를 JSON/XML로 제공하며, 개발계정 기본 트래픽은 1,000건이다.
- 고용24 OPEN-API는 인증키 기반이며 채용정보, 공채속보, 강소기업 등 고용 관련 API를 제공한다.
- 사람인 Job Search API는 공채속보 `bbs_gb=1`, 채용공고 검색, JSON/XML 응답을 제공한다.
- 잡알리오는 공식 공개 목록과 상세 원문을 제한적으로 확인하며, 별도 공공데이터 API 승인이 확보되면 API 어댑터로 보강한다.
- 서울특별시교육청 하이잡은 공식 공개 채용정보 목록과 POST 상세 원문을 확인하며, 서울 지역 특성화고 공고·매칭형 채용정보의 우선 소스로 둔다.
- 나라일터는 인사혁신처 공직 채용정보 공식 채널이므로 공식 API 또는 허용된 데이터 경로가 확인될 때 연결한다.

## 운영 점검

`assets/job-feed.json`의 `sourceStatus`를 보면 어떤 정보원이 연결되었고 어떤 정보원이 미연결인지 확인할 수 있다.

- `configured: false`: Secret 또는 공식 연계 정보가 아직 없음
- `configured: true`, `ok: false`: Secret은 있으나 어댑터 구현 또는 API 응답 점검 필요
- `itemCount`: 해당 소스에서 최종 등록 가능한 후보 수
- `trackHint`: 필기형, 면접형, 균형형 소스 구분
- `secretReadiness`: API 키, 공식 피드 URL, 어댑터 준비 상태. Secret 값이 아니라 이름과 상태만 표시

오류가 나면 공고 영역을 비워 두지 않고 “자동 수집 대기/점검 필요” 상태를 표시한다. 공식 공고 확인에 실패한 경우에도 공채 후보는 유지하되, 상세 확정이 아니라 확인 필요 상태로 표시한다.

현재 실행에서 공식 소스가 일시적으로 0건을 반환하면 직전 정상 공고를 임시 보존하고 `staleFallbackItems`와 `fallbackFromPreviousRun`으로 표시한다. 이는 순간적인 HTTP 오류 때문에 공채 목록이 비어 보이는 것을 막기 위한 안전장치이며, 다음 정상 수집 때 새 원문으로 교체한다.

홈페이지 공채 허브에는 전형 유형 요약과 별도로 첫날수집, 공식공고 확인, 누락점검, 원서마감 카운터를 표시한다. 이 카운터는 단순 홍보용 숫자가 아니라 수집기가 공채 첫날을 놓쳤는지, 공식 공고 더블체크가 부족한지, 마감 후 표시 상태가 맞는지 검수하는 운영 지표다.

## 신뢰성 검증

`tools/verify_platform_reliability.mjs`는 공채 피드와 홈페이지 연결 상태를 자동 점검한다. 기본 검증은 로컬 파일만 확인하고, 필요하면 라이브와 공식 원문 링크까지 확장한다.

- 기본: `node tools/verify_platform_reliability.mjs`
- 라이브 배포 확인 포함: `node tools/verify_platform_reliability.mjs --live`
- 공식 원문 URL 접속 확인 포함: `node tools/verify_platform_reliability.mjs --external-links`

검증 항목은 작업 폴더 안전성, Secret 노출 방지, GitHub Actions 하루 3회 실행 조건, 공채 피드 JSON 구조, 전형별 카운터 일치, 원서 마감 제목 처리, 첫날수집 음수 지연 방지, 공채 상세 2중확인 상태, 라이브 `gyo6.kr` 응답 상태를 포함한다. GitHub Actions에서도 피드 생성 후 이 검증을 통과해야 자동 커밋이 진행된다.
