# 특성화고 공채정보 자동 수집 시스템 계획

## 원칙

- 수동 등록은 기본 운영에서 제외한다.
- 공식 API 또는 공식 제휴 경로를 우선 사용한다.
- 사람인, 고용24/워크넷 등 공식 채용 데이터는 원문 링크, 마감일, 검증 시각을 함께 저장한다.
- API 키, 토큰, 접근키는 GitHub Actions Secrets에만 저장하고 저장소에는 기록하지 않는다.
- 비공식 크롤링은 약관, robots, 저작권, 개인정보 위험을 확인하기 전까지 기본 수집 경로로 사용하지 않는다.

## 1차 자동화 구조

`tools/fetch_vocational_jobs.mjs`가 하루 3회 실행되어 `data/job-feed.json`을 생성한다.

- 실행 시간: 09:10, 14:10, 19:10 KST
- 실행 위치: GitHub Actions
- 저장 방식: 정적 JSON 자동 갱신 후 커밋
- 표시 방식: `index.html`이 `data/job-feed.json`을 읽어 홈페이지 공채 허브에 표시

이 방식은 별도 서버 비용이 없고, GitHub Pages 캐시에도 비교적 안전하다.

## 공식 정보원 우선순위

1. 고용24 OPEN-API 공채속보
   - 고졸, 학력무관, 특성화고, 마이스터고, 학교장 추천 조건을 중심으로 조회한다.
   - Secret 이름: `WORK24_AUTH_KEY` 또는 `WORKNET_AUTH_KEY`

2. 사람인 채용공고 API
   - 공채속보(`bbs_gb=1`)와 고졸·특성화고·직업계고·마이스터고 키워드를 조합한다.
   - Secret 이름: `SARAMIN_ACCESS_KEY`

3. 잡코리아 공식 연계
   - 공개 API 또는 제휴 엔드포인트가 확인되면 `JOBKOREA_API_URL`, `JOBKOREA_API_KEY`로 연결한다.
   - 확인 전에는 무단 크롤링하지 않는다.

4. 지역·공공 청년일자리 API
   - 잡아바, 지자체 공공데이터, 공공기관 채용 소식은 공식 API와 이용조건 확인 후 추가한다.
   - Secret 예시: `JOBABA_API_URL`, `JOBABA_API_KEY`

## 자동 분류 기준

각 공고는 다음 필드로 정규화한다.

- 원문 출처와 원문 링크
- 기업명, 공고명, 지역, 고용형태
- 학력조건, 경력조건
- 마감일과 마감 상태
- 고졸·특성화고 적합도 점수
- 학교장 추천 여부
- 법률·지도자료 체크 태그
- 검증 시각

## 필터링 기준

유지한다.

- 고졸, 고등학교, 특성화고, 직업계고, 마이스터고, 졸업예정, 학교장 추천이 명확한 공고
- 학력무관이면서 신입·경력무관·공채 성격이 강한 공고
- 원문 링크와 마감일 확인이 가능한 공고

제외한다.

- 이미 마감된 공고
- 대졸 이상, 4년제 이상, 석사·박사 등 고졸 예정자와 맞지 않는 조건만 있는 공고
- 원문 링크가 없거나 채용 여부 확인이 어려운 공고

## 운영 점검

`data/job-feed.json`의 `sourceStatus`를 보면 어떤 정보원이 연결되었고 어떤 정보원이 미연결인지 확인할 수 있다.

- `configured: false`: Secret 또는 공식 연계 정보가 아직 없음
- `ok: false`: Secret은 있거나 실행했지만 API 응답 오류가 있음
- `itemCount`: 해당 소스에서 최종 등록 가능한 후보 수

오류가 나면 공고 영역을 비워 두지 않고 “자동 수집 대기/점검 필요” 상태를 표시한다.
